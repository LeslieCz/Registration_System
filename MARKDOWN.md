# DevSecOps CI Pipeline Prototype

Vulnerabilities related to course materials, such as the OWASP Top 10 web application risks, safe development techniques (NIST SSDF), and security monitoring principles,  were purposefully included in the mock application. Each code vulnerability corresponds to particular course modules, showing how automated SAST tools may identify actual security problems in a continuous integration pipeline.

## Features
- Automated pipeline triggered on code push
- Static Application Security Testing (SAST)
- Detection of insecure coding practices
- DevSecOps integration

## Tools Used
- GitHub Actions
- Python
- Bandit
